# Tech Stack: Recipes MCP Server\n\n## Core Language\n- **Rust:** The primary language for high-performance and safe data processing.\n\n## Frameworks and SDKs\n- **mcp-sdk-rust:** For implementing the Model Context Protocol (MCP) server interface.\n- **rust-recipe:** To leverage existing recipe parsing and data structures.\n- **recipe-scraper:** For extracting recipe information from diverse web sources.
- **scraper:** For HTML parsing and CSS selectors (used for King Arthur weight chart scraping).\n\n## Configuration & Data Formats\n- **Serde:** For seamless serialization and deserialization of all data structures.\n- **TOML:** Support for configuration files in \`.toml\` format.\n- **YAML:** Support for configuration files in \`.yaml\` or \`.yml\` format.\n- **JSON:** Support for configuration files in \`.json\` format.\n\n## CLI & Environment Variables\n- **Clap:** For robust command-line argument parsing and help generation.\n- **Dotenv / Envy:** For managing environment-based configuration and secrets.\n\n## Networking & IO\n- **Reqwest:** For fetching recipe web pages (likely a dependency for the scrapers).\n- **Tokio:** The asynchronous runtime for non-blocking IO operations.\n\n## Testing & Quality\n- **Cargo Test:** Standard Rust testing framework.\n- **Tarpaulin:** For code coverage reporting.
