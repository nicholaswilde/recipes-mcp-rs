# Product Guidelines: Recipes MCP Server\n\n## General Style\n- **Voice and Tone:** Clear, concise, and professional.\n- **Terminology:** Use consistent naming for internal tools, resources, and configuration keys.\n\n## Data Handling\n- **Privacy:** Do not log or store personal user data or private recipe URLs without explicit consent.\n- **Accuracy:** Prioritize high-fidelity parsing of ingredient measurements and preparation steps.\n- **Validation:** Sanitize all inputs from external web sources before processing.\n\n## Output Formatting\n- **Standardization:** All exported recipe data should follow a consistent schema (e.g., JSON-LD or a custom internal format).\n- **Readability:** Ensure that printed steps and ingredient lists are well-formatted for human consumption.\n\n## Configuration & Integration\n- **Convention Over Configuration:** Provide sensible defaults for all configuration options.\n- **Descriptive Errors:** Provide clear and actionable error messages for common issues (e.g., malformed URL, parsing failure).\n- **Compatibility:** Maintain compatibility with common MCP clients and host environments.

## MCP Tool Design
- **Unified and Action-Oriented Tools:** Related operations should be consolidated into unified tools using an `action` pattern (e.g., `manage_recipes`). This reduces token consumption and simplifies the toolset while maintaining full functionality and clarity through well-defined sub-actions.
